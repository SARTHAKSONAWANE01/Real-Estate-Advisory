Q3: DATABASE SCHEMA (MYSQL)
------------------------------------------------------------------------
Here is a proposed structure for the MySQL table that stores client inquiries:

CREATE TABLE inquiries (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) NOT NULL,
    phone VARCHAR(20) NOT NULL,
    package_name VARCHAR(100), -- E.g. 'Acquisition Advisory', 'Portfolio Management'
    message TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(50) DEFAULT 'New' -- Can be 'New', 'Contacted', 'Closed'
);

Suggestions:
  - Does this cover all fields you want to collect, or do you want to add fields 
    like "Budget Range", "Current Location", or "Investment Horizon"?

Example of how you can answer:
  "Let's add 'budget_range' and 'location' fields since real estate clients 
   have different budgets and locations."
